# KOL 挖掘功能设计

> 目标：从小红书话题数据中挖掘符合国际教育业务的用户，筛出可签约"为我们发声"的候选 KOL。
> v1 范围：**挖掘 + 候选名单**（轻量），层级聚焦**腰部 + KOC**。

## 一、数据现实（决定架构）

存量 2560 篇笔记里的作者信息：

| 字段 | 现状 |
|------|------|
| user_id / 昵称 / 头像 | ✅ 100% |
| 篇均互动（赞/评/藏）| ✅ 有 |
| 发文情感 | ✅ 已分析 |
| **粉丝数** | ❌ 全 0（搜索接口不返回，需 `get_user_info` 补）|

去重作者 1673 个，发过 ≥2 篇的 470 个。

**结论**：免费数据能"捞对人"（相关度+互动+情感），但没粉丝量级。粉丝数靠付费富化补，成本锁在 shortlist 规模内。

## 二、两级漏斗

```
① 候选挖掘（免费，聚合存量笔记）
   作者聚合 → 相关度+互动+情感 打分 → 候选池
   自家账号排除 / 竞品账号打标
        ↓ 人工圈 shortlist
② 富化（付费，仅对 shortlist）
   TikHub get_user_info → 补粉丝/认证/简介/地区
   三道成本锁：只查圈中人 · 结果永久缓存 · 每日调用上限
        ↓
   套「粉丝层级」筛选 → 最终名单 → 导出
```

## 三、账号归类规则

- **自家账号**（排除）：昵称含 `渊学通` 或 `英通`。
- **竞品账号**（打标，不排除）：命中词全部属于竞品词（`唯寻`/`犀牛`…），且非自家。
- 其余为**候选**。

## 四、综合分（0–100，分项透明）

```
fit_score = 0.40 · 相关度 + 0.35 · 互动 + 0.25 · 情感
相关度 = min(发文数/8,1)·70 + min(命中词数/3,1)·30
互动   = min(log10(篇均互动+1) / log10(5001), 1) · 100
情感   = 正面笔记占比 · 100
竞品账号：fit_score × 0.2（重罚，靠筛选器可隐藏）
富化后：按粉丝层级追加加分（腰部/KOC 最高）
```

互动量在免费阶段作为**影响力代理**——有真实赞评藏 ≈ 有受众。

## 五、数据模型 `kol_candidates`

```
user_id / nickname / avatar
note_count            话题下发文数
keywords_hit[]        命中的监控词
avg_engagement        篇均(赞+评+藏)
positive_rate         正面笔记占比
avg_sentiment_score
last_post_at
is_own / is_competitor
fit_score
── 富化后 ── fans_count / verified / bio / ip_location / enriched_at
── 人工 ──   status: candidate | shortlisted | rejected · remark
```

候选池由聚合实时算 + 人工态（status/remark/富化结果）单独持久化，合并展示。

## 六、接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/kol/candidates` | 列表+筛选（话题词/最低互动/情感/隐藏自家竞品/层级）|
| POST | `/api/kol/{uid}/enrich` | 付费富化（成本锁）|
| POST | `/api/kol/{uid}/status` | 置 shortlisted / rejected + remark |
| GET | `/api/kol/export` | 导出 CSV |

## 七、前端 `/dashboard/kol`

- 筛选条：话题词 · 最低篇均互动 · 情感 · 隐藏自家/竞品 ·〔粉丝层级—富化后〕
- 主表：昵称｜命中词｜发文｜篇均互动｜情感｜〔粉丝〕｜综合分｜状态｜操作
- 行展开：该作者话题下 top 笔记
- 操作：加入 shortlist · 富化 · 拒绝 · 导出

## 八、分期

- **v1（现在，免费可跑）**：挖掘聚合 + 打分 + 候选页 + shortlist/导出。富化按钮与缓存逻辑一并建好，TikHub 充值后即用。
- **二期**：`search_users` 主动扩池；签约流程跟踪；签约后投放效果追踪。
