# News Now 风格运营情报工作台布局设计

**日期：** 2026-05-26  
**范围：** 将 `/dashboard` 从左侧导航 + 纵向卡片流，升级为“来源导航 + News Now 式磁贴墙”的高密度情报总览。  
**阶段：** 前端布局重构；不新增后端来源，不改变采集链路。

---

## 目标

当前 `/dashboard` 的信息结构偏传统后台：左侧导航、中央列表、右侧辅助栏。它可读但不够像“情报台”，一屏内看不到多来源的相对热度和更新节奏。

这次设计目标是参考 News Now 的信息密度：

- 一屏扫到多个来源
- 每个来源用独立磁贴呈现
- 磁贴内部用榜单而不是大段摘要
- 点击来源后仍能进入更细的来源详情

最终效果不是娱乐新闻站，而是“国际教育官方情报雷达”。

---

## 选定方案

采用 **B：导航 + 磁贴墙**。

### 保留

- 左侧来源导航
- 现有 `/api/intel/overview`
- 现有 `/api/intel/sources/{source_key}`
- 当前官方来源 key：`ucas`、`university_site`、`exam_board`、`visa_policy`、`wechat_media`

### 改造

- 总览页主区改为多来源磁贴墙
- 来源详情页改为更紧凑的榜单阅读
- 大学官网同步状态从大状态卡压缩为顶部状态条
- 右侧辅助栏降级为小型统计，不抢主视觉

### 暂不做

- 真正的热度算法
- 已读状态持久化
- 拖拽排序
- 个性化来源订阅
- 新增后端数据字段

---

## 页面结构

### 总览页

总览页采用三层结构：

1. 顶部栏：产品名、轻量筛选、最近更新时间
2. 主区域：News Now 式来源磁贴墙
3. 右侧辅助栏：影响对象统计、同步健康度

磁贴墙是页面核心。每个磁贴代表一个来源或重点子来源：

- `UCAS`
- `海外大学官网`
- `Oxford`
- `Cambridge`
- `UCL`
- `Imperial`
- `考试局`
- `签证政策`
- `媒体公众号`

首版可以从现有 overview 数据派生，不要求后端一次性提供上述全部子来源。如果某个子来源没有数据，显示空状态或不渲染该磁贴。

### 来源详情页

来源详情页不再使用大卡片流，而是使用榜单化列表：

- 左侧或主区显示序号 + 标题 + 影响对象 + 发布时间
- 单条可展开或进入原文
- 保留 `去原文` 行为
- 大学官网页顶部显示同步状态条：正常数、被拦截数、失败数

来源详情页的目标是深读，不追求总览页那种多磁贴冲击力。

## 组件设计

### `DashboardShell`

负责整体页面骨架：

- 左侧导航
- 顶部栏
- 主内容区域
- 右侧辅助栏

它只管布局，不处理具体数据转换。

### `NewsNowOverview`

总览页主组件，输入 `IntelOverviewResponse` 和同步状态，输出磁贴墙。

职责：

- 把 overview sections 转成磁贴组
- 控制网格布局
- 处理空来源
- 传递磁贴点击事件

### `SourceTile`

单个来源磁贴。

显示内容：

- 来源图标或简称
- 来源名称
- 更新时间或状态
- 5-8 条榜单项
- 轻量状态按钮：刷新、收藏、更多（先做视觉）

磁贴颜色按来源区分，但保持低饱和：

- UCAS：蓝色
- 大学官网：绿色
- 签证政策：红棕色
- 考试局：灰蓝色
- 媒体公众号：灰色

### `RankedIntelRow`

磁贴内的一行榜单项。

显示内容：

- 序号
- 标题
- 小标签：本科 / 硕士 / 申请季 / 签证 / 奖学金
- 可选热度徽标

行内不展示长摘要。长摘要留给来源详情页。

### `SourceRankedPanel`

来源详情页主组件。

职责：

- 用紧凑榜单替代当前大卡片列表
- 保留原文跳转
- 展示影响对象和来源元信息
- 支持空状态

## 数据映射

不改后端响应结构，首版只在前端做派生。

### 总览磁贴

从 `IntelOverviewResponse.sections` 中读取：

- `source_key`
- `source_label`
- `total_items`
- `preview_items`

每个 section 生成一个一级磁贴。

### 大学子磁贴

从 `university_site` 的 items 中按 `school_name` 派生：

- `Oxford`
- `Cambridge`
- `UCL`
- `Imperial`

如果某个学校没有 item，但 sync report 存在，则可以展示状态磁贴。这样 `Oxford blocked` 仍然可见。

### 同步状态

继续使用：

- `/api/intel/sources/university_site/sync-status`
- `/api/intel/sources/university_site` 返回中的 `sync_reports`

总览页只展示压缩后的健康度，不展开全部错误。

## 交互规则

### 顶部筛选

首版提供静态分段控件：

- `最新`
- `最热`
- `待读`

默认选中 `最新`。另外两个先只做禁用或视觉占位，避免假功能。

### 点击磁贴

点击来源磁贴后切换到对应来源详情页：

- UCAS -> `ucas`
- 海外大学官网 / 子学校 -> `university_site`
- 签证政策 -> `visa_policy`
- 考试局 -> `exam_board`
- 媒体公众号 -> `wechat_media`

### 原文跳转

榜单项点击主标题可打开原文，或保留单独的 `去原文` 链接。首版选择标题打开原文，减少按钮噪音。

---

## 响应式

### 桌面端

- 左侧导航固定
- 主区使用 3-4 列磁贴
- 右侧辅助栏保留

### 平板端

- 左侧导航压缩为顶部横向来源栏
- 磁贴 2 列
- 右侧辅助栏移动到主区底部

### 手机端

- 单列磁贴
- 每个磁贴最多显示 5 条
- 顶部筛选保持横向滚动

---

## 视觉原则

1. 背景保持深色，不使用大面积纯黑以外的装饰
2. 磁贴颜色服务于来源识别，不做过强渐变
3. 卡片圆角控制在 8-10px
4. 标题优先，摘要后置
5. 一屏内优先展示“发生了什么”，不展示长解释
6. 不使用营销 hero，不把页面做成 landing page

---

## 风险

### 信息密度过高

解决：每个磁贴限制 5-8 条，长摘要放到来源详情页。

### 来源颜色过杂

解决：使用低饱和来源色，并保留统一深色底。

### 数据不足导致页面空

解决：空来源显示轻量空态；大学官网可用 sync report 生成状态磁贴。

### 看起来像娱乐新闻站

解决：文案、图标和颜色保持克制，标签聚焦申请、签证、奖学金、学校动态。

---

## 验收标准

1. `/dashboard` 默认总览页展示 News Now 式磁贴墙
2. 左侧来源导航仍可正常切换
3. 每个来源磁贴至少能展示来源名、数量、榜单项
4. 大学官网能展示学校级磁贴或同步状态
5. 来源详情页使用紧凑榜单，不再是大卡片流
6. `npm run lint` 和 TypeScript 检查通过
7. 桌面宽屏下首屏能看到至少 6 个来源磁贴
8. 移动端不出现文字溢出或卡片重叠
